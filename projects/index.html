Board Game Hunter

Make hunting down your favourite out of print board game (less) painful

This is my current project. The project has a sometimes-used project page on github here. Its written in Clojure/Clojurescript using the Luminus Framework. You can learn about the framework via the accompanying book "Web Development with Clojure", found here.

Current Status

The search is functional for bgg's 'classic' market listings. Auctions are WIP and, although I love it, BGG has once again made my life difficult by adding a new market-to-replace-all-markets. So guess I'll be working on that as well.

Goals

  1. Search all of bgg in one location

    We were almost there until BGG added a new market. Further investigation is needed to see what is required from an api-access standpoint.

  2. Aggregate search with notifications

    One off searches are nice, but the ability to save a collection and get pinged via PWA notification on your phone would be even nicer

  3. Integration with other sites

    BGG market hunts are great, but sometimes other board game sites like GameNerdz have fantastic deals on new or ding-and-dent copies of games, factor in their cheap or even free shipping and the used market all of the sudden does not look as great (from a price point perspective, environmentally still all about buying used).

Freelancer's Calendar

The Google Calendar replacement you didn't know you needed (still w.i.p.)

This was my main pre-full-time project. Its written in React, and uses a Node.js backend connected to a MongoDB instance. There are no class components, and a lot of hooks. Code viewable here, the live site is hosted here and an embed of the site may be found below.

Current Status

The site is functional! You log in through google, create and manage events, then view them in weekly or monthly views.

NOTE: I have a side goal of making my own web components for input, so I just put in a placeholder form for editing events. It assumes 24hr time and YYYY/MM/DD formatted dates.

Goals

  1. Reach feature parity with the core of Google Calendar

    Right now, that means allowing users to have multiple calendars, the ability to share events/calendars with each other and a few additional calendar views.

  2. Add features unique to the Freelancer's Calendar

    The most important feature enhancement is the ability to set flexible calendar items, and move them as a group. With these features, a meeting can run long without making you want to throw your whole calendar away for the rest of the day.

  3. Integration with other calendars (iCal/gCal)

    Event importing/exporting is a given, but I'd like the ability to keep the calendars in sync. Research into API's and license agreements is necessary before moving forward.

Nand2Tetris

Taking the term 'full-stack' too far

This is probably my favorite project right now. The idea is to build a heavily simplified computer, write a language for it, then an operating system, and eventually a user app (tetris).

  1. Use HDL to specify hardware components for a computer

    In other words, start with just NAND gates and build up the logic for a simple computer with a register based cpu and ram.

  2. Write a linker for a simple assembly language targeting the aforemention hardware

    A description of the assembly language is given as a start point. My code for the linker was written in Java, and is shown in a gist embedded below.

  3. Write a VM translator for a given IR to our assembly language

    The system language compiles to a VM. The first step is to map the VM language to the hardware's assembly language.

  4. Write a compiler for a object-based language to the VM

    The language has basic typing and classes but no inheritance. A small standard library, including screen utilities is also written.

  5. Write an Operating System in the new language

    This includes a runtime, as well as software abstractions for hardware (screen, keyboard) and better memory allocation utilities.
